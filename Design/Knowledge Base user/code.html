Understanding Data Synchronization in Neeflow

calendar_month Updated 2 days ago
•
schedule 8 min read

Neeflow uses a proprietary real-time engine to ensure your data stays consistent across all connected platforms. This guide covers the architectural overview, configuration steps, and best practices for conflict resolution.

Architectural Overview

The sync engine operates on a distributed event-mesh. When a change is detected in any connected source system, Neeflow generates a "Delta Event" which is then propagated to all relevant endpoints within milliseconds.

Fig 1.0: The Neeflow Multi-Directional Sync Topology

Configuring Your First Sync

Follow these steps to establish a secure synchronization link between your primary database and the Neeflow cloud instance:

  1. Navigate to Settings > Data Management in your dashboard.
  2. Select the "Add New Source" button and choose your provider.
  3. Authenticate using our OAuth2 flow or provide an encrypted API Key.
  4. Define your Sync Frequency (Real-time, Hourly, or Daily).
info

Pro Tip

For high-traffic applications, we recommend using a 5-minute polling interval instead of true real-time to optimize API quota usage on legacy systems.

Conflict Resolution

In instances where two sources update the same record simultaneously, Neeflow defaults to "Last Write Wins" (LWW). However, you can customize this logic using our Conflict-Policy header.
